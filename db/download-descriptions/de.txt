ChangeLog-AROS:
    Änderungsprotokolldatei für die AROS-Kern-Quellcodedateien.
ChangeLog-contrib:
    Änderungsprotokolldatei für die Quellcodes der beigestellten Programme.
ChangeLog-docs:
    Änderungsprotokolldatei für die Dokumentation.    
source:
    AROS-Kern-Quellcodedateien.
docs-source:
    Dokumentation Quelldateien.
contrib-source:
    Quellcode der beigestellten Programme.
docs-html:
    Dokumentation im HTML-Format. Enthält alles, was auf der Webseite
    vorhanden ist, aufbereitet zum Offline-Lesen.
amiga-m68k-system:
    Die native Version für m68k Amigas oder UAE. Diese Version ist binärkompatibel
    mit AmigaOS. Dieses Archiv enthält ROM-Speicherabbilder und alle AROS
    System-Binärdateien, welche aus dem Subversion-Verzeichnis erstellt wurden.
amiga-m68k-boot-floppy:
    Die native Version für m68k Amigas oder UAE. Diese Version ist binärkompatibel
    mit AmigaOS. Dieses Archiv enthält ROM-Speicherabbilder und ein
    Boot-Floppy-Speicherabbild.
amiga-m68k-boot-iso:
    Die native Version für m68k Amigas oder UAE. Diese Version ist binärkompatibel
    mit AmigaOS. Dies ist das CDROM-ISO-Image.
darwin-i386-system:
    Die gehostete Version für Darwin und MacOS x für die i386-Platform.
    Sollte auf allen Intel-MacOS-X-Versionen ab 10.4 arbeiten.
darwin-ppc-system:
    Die gehostete Version für Darwin und MacOS x für die PowerPC-Platform.
    Sollte auf allen MacOS-X-Versionen arbeiten.
darwin-x86_64-system:
    Die gehostete Version für Darwin und MacOS x für die x86-64-Platform.
    Sollte auf allen Intel-MacOS-X-Versionen ab 10.6 arbeiten.
freebsd5-i386-system:
    Die gehostete Version für FreeBSD/i386 5.x. Beachte, dass
    sie nicht auf FreeBSD 4.x-Systemen läuft.
freebsd-i386-system:
    Die gehostete Version für FreeBSD/i386 5.x. Beachte, dass
    sie nicht auf FreeBSD 4.x-Systemen läuft.
linux-i386-system:
    Die gehostete Version für Linux/i386. Dieses Archiv enthält
    alle AROS-System-Binärdateien aus dem Subversion-Archiv.
mingw32-x86_64-system:
    The hosted version for Microsoft Windows on x86_64 platform.
mingw32-i386-system:
    The gehostete Version für Microsoft Windows auf der i386-Plattform.
    Sie wurde entwickelt und kompiliert unter Windows XP, sollte jedoch unter
    allen 32-bit Windowsversionen ab Windows 98 laufen.
linux-x86_64-system:
    Die gehostete Version für Linux/x86_64. Dieses Archiv enthält
    alle AROS-System-Binärdateien aus dem Subversion-Archiv.
linux-ppc-system:
    Die gehostete Version für Linux/PowerPC. Dieses Archiv enthält alle
    AROS-System-Binärdateien aus dem Subversion-Archiv.
linux-arm-system:
    Die gehostete Version für Linux/ARM. Dieses Archiv enthält die
    AROS-System-Binärdateien aus dem Subversion-Archiv. Beachte bitte dass
    diese Version mindestens eine ARMv6-CPU mit VFP-Einheit benötigt.
pc-i386-boot-floppy:
    Die native Version für i386-PC-AT-Computer und Kompatible.
    Dieses Archiv enthält das Boot-Diskettenimage, welches Du
    auf eine Diskette schreiben kannst um AROS davon zu booten.
pc-i386-boot-iso:
    Die native Version für i386-PC-AT-Computer und Kompatible.
    Dies ist das boot-fähige CDROM-ISO-Image, welches alle AROS-
    System- und beigestellten Programme enthält.
pc-x86_64-boot-iso:
    Die native Version für 64-Bit (x86_64, amd64) Computer.
    Dies ist das bootbare CDROM ISO-Image, welches alle
    AROS-Systemprogramme und beigesteuerte Programme enthält.
pc-x86_64-smp-boot-iso:
    This is a highly experimental SMP build for pc-x86_64.
    Only recommended for Developers. This is a bootable CD-ROM ISO image.
sam440-ppc-boot-iso:
    Die native Version für Sam440EP, Sam440EP Flex und Sam460ex Computer.
    Dies ist das bootfähige ROM-Speicherabbild einschließlich
    ub2lb (Parthenope) SLB. Es enthält alle AROS System- und
    beigesteuerte Programme.
efika-chrp-ppc-boot-iso:
    Die native Version für PowerPC Efika Computer. Dies ist das
    bootfähige CDROM ISO-Speicherabbild einschließlich ub2lb für
    "open firmware". Es enthält alle AROS System- und beigesteuerte
    Programme.
raspi-armhf-system:
    The native version for ARMv6 based Raspberry Pi computers.
    Contains all files needed to boot from a FAT formatted sd-card.
raspi-armhf-contrib:
    This archive contains all the contributed programs,
    compiled for the ARM architecture, for use with the
    RasPi/ARM port (using hard-float ABI). Please note that this
    version requires at least an ARMv6 CPU with VFP unit.
raspi-armhf-ports:
    This archive contains applications and games which are built
    from external sources for the armhf architecture (both hosted
    and native). Because of its size the archive will only be updated
    from time to time on the snapshots page.
pc-i386-contrib:
    Dieses Archiv enthält alle beigesteuerten Programme für die
    i386-Architektur. Sie funktionieren sowohl für hosted (z.B. i386-linux)
    als auch für native (i386-pc).
pc-x86_64-contrib:
    Dieses Archiv enthält alle beigesteuerte Programme für die
    native Version der X86_64-Architektur.
linux-ppc-contrib:
    Dieses Archiv enthält alle beigesteuerten Programme für die
    PowerPC-Architektur für die Linux/PowerPC-Portierung.
sam440-ppc-contrib:
    Dieses Archiv enthält alle beigesteuerte Programme für die
    PowerPC-Architektur zur Verwendung mit dem Sam440EP-Port.
pc-i386-sdk:
    Dateien und Werkzeuge um Deine eigenen AROS-Programme
    auf Deiner Maschine zu erstellen. GCC und die Binutils
    müssen so konfiguriert sein, dass sie Programme im
    i386-elf-Format erzeugen.
linux-ppc-sdk:
    Dateien und Werkzeuge um Deine eigenen AROS-Programme
    auf Deiner Maschine zu erstellen. GCC und die Binutils
    müssen so konfiguriert sein, dass sie Programme im
    ppc-elf-Format erzeugen.
linux-arm-sdk:
    Dateien und Werkzeuge um Deine eigenen AROS-Programme auf Deiner
    bevorzugten Maschine zu erstellen. GCC und die Binutils müssen so
    konfiguriert sein, dass sie Programme im arm-elf-Format erzeugen.
sam440-ppc-sdk:
    Dateien und Werkzeuge um Deine eigenen AROS-Programme auf Deiner
    bevorzugten Maschine zu erstellen. GCC und die Binutils müssen so
    konfiguriert sein, dass sie Programme im ppc-elf-Format erzeugen.
pc-x86_64-sdk:
    Dateien und Werkzeuge um Deine eigenen AROS-Programme
    auf Deiner Maschine zu erstellen. GCC und die Binutils
    müssen so konfiguriert sein, dass sie Programme im
    x86_64-elf-Format erzeugen.

Linux-gnu-pc-i386-toolchain:
    Linux GNU crosscompiler for AROS i386 target
Linux-gnu-pc-x86_64-toolchain:
    Linux GNU crosscompiler for AROS x86_64 target
Linux-gnu-sam440-ppc-toolchain:
    Linux GNU crosscompiler for AROS PPC target
Linux-gnu-raspi-arm-toolchain:
    Linux GNU crosscompiler for AROS ARM target
Linux-gnu-raspi-armhf-toolchain:
    Linux GNU crosscompiler for AROS ARM target
Linux-gnu-mingw32-i386-toolchain:
    Linux GNU crosscompiler for AROS mingw32 i386 target
